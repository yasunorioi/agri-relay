# agri-relay 操作マニュアル

**対象機器**: agri-relay（CCMリレーノード）  
**対象読者**: 農家・施設管理者  
**作成日**: 2026-04-13

---

## 1. はじめに

このマニュアルは、温室・農業施設向けリレー制御装置 **agri-relay** の設定・操作方法を説明します。

agri-relay は Wi-Fi で接続し、ブラウザから設定できます。専門的な知識がなくても、このマニュアルに沿って操作できます。

### 接続方法

1. agri-relay と同じ Wi-Fi ネットワークにパソコン・スマートフォンを接続する
2. ブラウザのアドレス欄に以下を入力する  
   `http://uecs-ccm-01.local`  
   ※ mDNS が使えない場合は IP アドレスを直接入力してください

> **Windows をお使いの方**: Windows 11 は mDNS に対応しているため、追加ソフトなしでそのまま接続できます。  
> 繋がらない場合は、Wi-Fi の設定で「ネットワークプロファイル」を **「プライベート」** に変更してください。

### 画面の共通ルール

- 設定を変更したら **「保存」ボタン** を押してください。押さないと変更が反映されません
- 画面上部のナビリンクをクリックすると各設定ページに移動できます
- 「再起動不要」と書かれた設定は保存すると即時に反映されます
- 「保存して再起動」の設定は保存後に装置が再起動します（約10秒）

## 2. リレー割当表

agri-relay には CH1〜CH7 のリレー（スイッチ）があります。各チャンネルの接続先は以下の通りです。

| チャンネル | 接続機器 | 動作 |
|-----------|---------|------|
| CH1 | 東側窓 | 開ける（モーター正転） |
| CH2 | 東側窓 | 閉める（モーター逆転） |
| CH3 | 西側窓 | 開ける（モーター正転） |
| CH4 | 西側窓 | 閉める（モーター逆転） |
| CH5 | 灌水バルブ | 灌水 ON/OFF |
| CH6 | 換気扇（ファン A） | 換気 ON/OFF |
| CH7 | 換気扇（ファン B） | 換気 ON/OFF |

> **ご注意**: 側窓は「開ける」と「閉める」で別チャンネルを使います。  
> 例えば東側窓を開けるには CH1 を ON にします。  
> 開閉操作は同時に行わないように制御されています。

## 3. 各設定ページの説明

### 3-1. ダッシュボード（トップページ `/`）

装置の現在の状態を一覧で確認できます。設定の変更は行えません。

**表示内容**:

| 項目 | 内容 |
|:------------------|:----------------------------------------------|
| Node / FW / 起動時間 | 装置の識別名・ソフトウェアバージョン・起動からの経過時間 |
| IP アドレス / mDNS | ネットワーク上の住所 |
| センサー接続状況 | 温度センサー・日射センサー等の接続 OK / NG |
| リレー状態 | CH1〜CH8 の ON/OFF 状態（手動でも切替可） |
| デジタル入力 | 外部センサー・スイッチの ON/OFF 状態 |
| センサー値 | 現在の温度・湿度・日射量・CO2 濃度 |
| 温室制御 / 灌水 / 保護 | 各制御の動作状態を簡易表示 |

**手動操作**: リレー行の「ON」「OFF」ボタンで手動切替できます。自動制御より手動操作が優先されます。

### 3-2. CCM 割当設定（`/ccm`）

各リレーチャンネルに UECS-CCM（農業用通信プロトコル）のアクチュエータ種別を割り当てます。  
UECS を使っていない場合はこの設定は不要です。

**画面の説明**:

設定画面には CH1〜CH8 の行が並んでいます。各行で以下を設定します。

| 項目 | 説明 |
|:------------------|:----------------------------------------------|
| CCM タイプ | このチャンネルが制御する機器の種類。接続機器に合わせて選択 |
| 部屋 | UECS の部屋番号。自施設の部屋番号を入力（例: 1） |
| リージョン | UECS のリージョン番号。通常は 1 |
| 順序 | CCM コマンドの実行順序。通常は 1〜7 を順に設定 |
| 優先度 | 複数コマンド競合時の優先順位。通常は 1 |
| WDT(s) | ウォッチドッグタイム（秒）。この秒数以上コマンドが来ないと自動 OFF。0 で無効 |
| DI リンク | このリレーと連動させるデジタル入力チャンネル。通常は (none) |

**一括設定**: 「部屋」「リージョン」欄を入力して「全適用」ボタンを押すと、全チャンネルに同じ値を設定できます。

> **変更は即時反映されます（再起動不要）。**

### 3-3. 温室制御（`/greenhouse`）

温度に応じてリレーを自動制御します。側窓や換気扇を温度比例で動かす設定です。

#### 温室制御ルール

最大 4 つのルールを設定できます。各ルールで以下を設定します。

| 項目 | 説明 |
|:------------------|:----------------------------------------------|
| 有効チェック | このルールを使う/使わない。チェックを入れると有効 |
| CH | 制御するリレーチャンネル。例: CH3（西側窓 開）など |
| センサー | 温度を測るセンサーの選択。SHT40（室内）か DS18B20 |
| Open(C) | この温度以上でリレーが動き始める。例: 25℃ |
| Full(C) | この温度以上でリレーが 100% 稼働。例: 30℃ |
| Cycle(s) | 比例制御の周期（秒）。60 秒が目安 |
| Curve | 開度の変化カーブ。通常は Linear（直線） |

> **設定例（東側窓を温度比例で開ける場合）**:  
> CH=CH1、センサー=SHT40、Open=25℃、Full=30℃、Cycle=60s  
> → 25℃で開き始め、30℃でフル稼働になります

**カーブモードの説明**:

| モード | 動作 |
|:------------------|:----------------------------------------------|
| Linear（直線） | 温度に比例して均等に開く（通常はこちらを選択） |
| Table | 温度ごとの開度を手動で細かく設定 |
| Sigmoid | 中間温度で急激に変化するカーブ |
| Exponential | 高温になるほど急激に開くカーブ |

#### 側窓開度制御

時間ベースで側窓の開度（開き具合）を制御します。  
開方向・閉方向それぞれの全動作時間（秒）を設定し、開度%に比例した時間だけリレーを動作させます。

| 項目 | 説明 |
|:------------------|:----------------------------------------------|
| Open CH | 窓を「開ける」方向のリレーチャンネル |
| Close CH | 窓を「閉める」方向のリレーチャンネル |
| Limit DI | リミットスイッチ入力。本機では未使用のため「-」のまま |
| Open(s) | 全閉→全開にかかる秒数（例: 60秒） |
| Close(s) | 全開→全閉にかかる秒数（例: 60秒） |

> **設定例**: Open(s)=60、Close(s)=60 の場合、50%開度の指示で開方向に 30 秒間リレーを動作させます

### 3-4. 日射灌水（`/irrigation`）

太陽の光の積算量（日射量）に応じて自動で灌水します。  
晴れた日は日射量が多いので多く灌水し、曇りの日は少なく灌水します。

> **必要機器**: ADS1110 + PVSS-03 日射センサー（I²C Grove 接続）

#### 基本設定

| 項目 | 説明 |
|:------------------|:----------------------------------------------|
| 有効 | この灌水ルールを使う/使わない。チェックを入れると有効 |
| リレー CH | 灌水バルブのチャンネル。例: CH5 |
| モード | タイマーモード / デューティモード。最初はタイマーモードを推奨 |
| 閾値(MJ/m²) | この積算日射量に達したら灌水開始。0.3〜1.0（環境に合わせて調整） |
| Min W/m² | この日射量以下では灌水しない（雨の日除外）。50〜100 が目安 |

#### タイマーモード（モード=0）

積算日射量が閾値に達したら、一定時間だけ灌水します。

| 項目 | 説明 |
|:------------------|:----------------------------------------------|
| 継続時間(s) | 灌水する時間（秒） |
| 排液停止(s) | 灌水後に排液を確認するための待機時間 |

> **設定例**: 閾値=0.5 MJ/m²、継続時間=300s（5分）  
> → 積算日射量が 0.5 MJ/m² に達するたびに 5 分間灌水します

#### デューティモード（モード=1）

排液センサーの値をフィードバックして灌水量を自動調整します。高度な制御方式です。

| 項目 | 説明 |
|:------------------|:----------------------------------------------|
| 周期(s) | 灌水 ON/OFF を繰り返す周期 |
| Init% | 初回灌水のデューティ比（初期値） |
| Min%/Max% | デューティ比の下限・上限 |
| Step% | 1 回の調整ステップ量 |
| 排液 Lo%/Hi% | 排液率の目標範囲（この範囲に収まるよう自動調整） |

#### センサーキャリブレーション

| 項目 | 説明 |
|:------------------|:----------------------------------------------|
| Flow DI | 流量センサーのデジタル入力チャンネル |
| mL/pulse | 流量センサー 1 パルスあたりの流量（mL） |
| Drain mL/tip | 排液チップバケット 1 回あたりの排液量（mL） |

### 3-5. 保護制御（`/protection`）

ハウス内の環境を守るための保護機能です。3 つの保護機能があります。

#### 結露対策（Dew Prevention）

日の出前に換気ファン・ヒーターを動かして、朝の結露を防ぎます。

| 項目 | 説明 |
|:------------------|:----------------------------------------------|
| 有効 | 結露対策を使う/使わない。チェックを入れると有効 |
| 緯度 | 施設の緯度（日の出時刻の計算に使用）。例: 43.0 |
| 経度 | 施設の経度。例: 141.0 |
| タイムゾーン(UTC+) | 時間帯の設定。日本は 9 |
| 日の出前(分) | 日の出の何分前からファンを動かすか。30〜60 分が目安 |
| 日の出後(分) | 日の出の何分後までファンを動かすか。30〜60 分が目安 |
| ファンリレー | 換気ファンのチャンネル。例: CH6 または CH7 |
| ヒーターリレー | ヒーターのチャンネル（オプション）。使わない場合は「-」 |
| 側窓スロット | 結露対策と連動させる側窓スロット。使わない場合は「-」 |
| 側窓開度% | 結露対策中に側窓を開ける割合。10〜20% が目安 |
| 終了時開度% | 結露対策終了時の側窓開度。通常は 0%（全閉） |
| 低温ガード(℃) | この温度以下では側窓を開けない。5〜10℃ が目安 |

> **設定例**: 緯度=43.06、経度=141.35（北海道）、タイムゾーン=9、日の出前=30分、日の出後=30分、ファン=CH6  
> → 毎朝、日の出の 30 分前から 30 分後まで換気ファン A を動かします
>
> **側窓連動**: 側窓スロット=Slot1、開度=10%、低温ガード=5℃ に設定すると、  
> 結露対策中に側窓を 10% 開けて換気効率を上げます。気温が 5℃ 以下のときは側窓を開けません

#### 温度急変対策（Rate Guard）

気温が急激に上昇したときにファンを稼働させます。  
日差しが急に強くなってハウス内が急激に暑くなる際の対策です。

> **発動条件: 1 時間に 5℃以上の気温上昇**（急上昇を検知したら即座にファンを稼働）

| 項目 | 説明 |
|:------------------|:----------------------------------------------|
| 有効 | 温度急変対策を使う/使わない。チェックを入れると有効 |
| 閾値(C/min) | 温度変化率の検知感度（1 分あたりの変化℃）。環境に合わせて調整 |
| センサー | 温度を測るセンサーの選択。SHT40（室内）か DS18B20 |
| ファンリレー | 稼働させるファンのチャンネル。例: CH6、CH7 など |
| 保持時間(s) | ファンを稼働し続ける時間（秒）。120〜300 秒が目安 |

> **動作の流れ**: 気温急上昇を検知 → ファンを稼働 → 保持時間が経過し気温変化が落ち着いたら停止

#### CO2 ガード（CO2 Guard）

CO2 濃度が低くなりすぎたときに換気を行います。  
ハウス内の CO2 が不足すると植物の光合成が低下します。

> **必要機器**: SCD41 CO2 センサー

| 項目 | 説明 |
|:------------------|:----------------------------------------------|
| 有効 | CO2 ガードを使う/使わない。チェックを入れると有効 |
| 閾値(ppm) | この濃度以下になったら換気。例: 200 ppm |

動作設定（最大 4 アクション）:

| 項目 | 説明 |
|:------------------|:----------------------------------------------|
| リレー CH | 動作させるリレーのチャンネル |
| 持続時間(s) | リレーを ON にする時間（秒） |

> **設定例**: 閾値=200ppm、アクション1=CH6 で 60s、アクション2=CH7 で 60s  
> → CO2 が 200ppm 以下になったらファン A・B をそれぞれ 60 秒動かします

### 3-6. ネットワーク設定（`/config`）

Wi-Fi ネットワークへの接続設定と機器識別情報を設定します。

> **注意**: この設定を保存すると装置が再起動します（約 10 秒）

| 項目 | 説明 |
|:------------------|:----------------------------------------------|
| node_id | 機器の識別 ID（半角英数字）。例: `uecs-ccm-01` |
| node_name | 機器の表示名（わかりやすい名前）。例: `東ハウス中継器` |
| IP アドレス | 固定 IP の場合に入力（空白で DHCP 自動取得）。例: `192.168.1.100` |
| Subnet | サブネットマスク。例: `255.255.255.0` |
| Gateway | ゲートウェイ（ルーターのアドレス）。例: `192.168.1.1` |
| DNS | DNS サーバーのアドレス。例: `192.168.1.1` |
| Hostname | mDNS のホスト名。例: `uecs-ccm-01` |
| mDNS 有効 | `uecs-ccm-01.local` でアクセスできるようにする。チェックを入れると有効 |

> **IP アドレスを固定する場合**: IP・Subnet・Gateway・DNS を全て入力してください。  
> DHCP（自動取得）の場合は IP 欄を空白のままにしてください。

### 3-7. ファームウェア更新（`/ota`）

装置のソフトウェア（ファームウェア）を新しいバージョンに更新します。  
通常の運用では操作不要です。

**手順**:

1. 「ファイルを選択」をクリックして `.bin` ファイルを選ぶ  
   （ファームウェアファイルは提供者から受け取ってください）
2. 「アップロード＆書込み」ボタンをクリック
3. 進捗バーが 100% になったら完了
4. 装置が自動的に再起動します（約 10 秒）

> **注意**:  
> - 更新中は電源を切らないでください  
> - 更新失敗のメッセージが出た場合は、もう一度手順 1 からやり直してください  
> - 現在のバージョンは画面上部に表示されています

## 4. よくある制御パターン

実際によく使う設定の組み合わせ例を紹介します。

### パターン 1: 結露防止（毎朝換気+側窓）

日の出前から換気ファンを動かし、側窓も少し開けて朝の結露を防ぎます。

**設定箇所**: 保護制御ページ「結露対策」  
- 日の出前: 30〜60 分  
- 日の出後: 30 分  
- ファンリレー: CH6（換気扇 A）  
- 側窓スロット: Slot 1、開度 10%、低温ガード 5℃  
  → 気温 5℃ 以上のときだけ側窓を 10% 開けて換気を補助します

---

### パターン 2: 高湿度時の換気

湿度 100% に近づいたときに換気で湿度を下げます（病気防止）。

**設定箇所**: 温室制御ページ「温室制御ルール」  
- センサー: SHT40（温度・湿度）  
- ルール 1: CH6 → 高温時に換気扇 A を稼働  
- ルール 2: CH7 → 高温時に換気扇 B を稼働  
- ルール 3/4: CH1〜CH4 → 必要に応じて側窓も開ける

---

### パターン 3: CO2 不足時の換気

ハウス内の CO2 が 200ppm 以下になったら換気します。

**設定箇所**: 保護制御ページ「CO2 ガード」  
- 必要機器: SCD41 センサー（別途接続が必要）  
- 閾値: 200 ppm  
- アクション: CH6・CH7 を 60 秒ずつ動作

---

### パターン 4: 低温時の側窓ロック

気温が 10℃以下のときは側窓を開かず、ファンのみで換気します。

**設定箇所**: 温室制御ページ「温室制御ルール」  
- ルール 1/2（側窓用）: Open(C)=10℃、Full(C)=15℃  
  → 10℃未満ではリレーが動かない（側窓が開かない）  
- ルール 3/4（ファン用）: 低温でも換気扇は動くように別途設定

---

### パターン 5: 温度急変対策（Rate Guard）

日差しが急に強くなってハウスの温度が急上昇したとき、ファンで冷やします。

**発動条件**: **1 時間に 5℃以上の気温上昇**を検知したら発動

**設定箇所**: 保護制御ページ「温度急変対策」  
- 有効: チェックを入れる  
- センサー: SHT40  
- ファンリレー: CH6 または CH7（または両方をルールで設定）  
- 保持時間: 120〜300 秒（2〜5 分）

> ファンが稼働して保持時間が過ぎると、気温変化が落ち着いたことを確認してから停止します。

## 付録: よくある質問

**Q. 設定を保存したのに変わらない**  
A. 「保存」ボタンを押した後、画面に「OK」が表示されているか確認してください。表示されていない場合はネットワーク接続を確認してください。

**Q. 機器のアドレスがわからない**  
A. 初期設定では `http://uecs-ccm-01.local` でアクセスできます。mDNS が使えない環境では、IP アドレスを Wi-Fi ルーターの管理画面から確認してください。

**Q. 側窓が動かない**  
A. ダッシュボードでリレーの ON/OFF 状態を確認してください。温室制御ルールが有効になっているか、Open/Full 温度の設定が現在の温度と合っているか確認してください。

**Q. 灌水が始まらない**  
A. 日射センサー（ADS1110）が接続されているか確認してください。ダッシュボードの「Solar」欄に値が表示されていれば接続できています。積算量が閾値に達すると灌水が始まります。

**Q. ファームウェア更新に失敗した**  
A. `.bin` ファイルが正しいファームウェアか確認してください。ファイルサイズが 8MB 以下であることも確認してください。
